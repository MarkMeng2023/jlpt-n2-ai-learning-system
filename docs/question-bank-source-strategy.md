# JLPT N2 题库来源与知识点验证策略

## 目标与发布原则

本方案把“题型正确”“知识点可信”“题目可使用”“题目质量合格”视为四件需要分别验证的事。AI 不能用自己的输出证明自己的正确性；一个网页可公开访问，也不代表其题目正文可被复制或改编。

当前扩题门禁为 **HOLD**：先完成现有知识点来源登记和题目解析复核，再进入 300 题阶段。自动基线报告位于 [`reports/question-bank-quality.md`](../reports/question-bank-quality.md)。

## 题库来源优先级

### P0：用户明确提供且有使用权限的材料

- 用户个人拥有并明确授权录入的真题、教材、讲义或笔记可以进入审核流程。
- 登记材料名称、版本、页码或题号，以及授权范围。
- 用户提供不等于版权自动清除；若将题库公开发布，需要再次确认公开传播或改编权限。

### P1：JLPT 官方公开资料

- 优先用于确认 N2 的科目、题型和测量目标。
- 官方公布的 N2 项目包括词汇的语境规定、近义改写和用法，文法形式判断、句子组成和文章文法，以及多种阅读与听力任务。[官方题型构成](https://www.jlpt.jp/e/guideline/testsections.html)与 [N2 测试项目目的](https://www.jlpt.jp/e/guideline/pdf/n2_e.pdf)是题型分类的最高优先依据。
- [官方样题与练习册页面](https://samplequestions.jlpt.jp/e/samples/sampleindex.html)优先用于理解出题形式，但不默认允许复制题目正文。
- JLPT 官方网站说明其内容受版权保护，并特别提示部分样题含第三方作品，需要另行授权。[JLPT Site Policy](https://www.jlpt.jp/e/policy.html)

### P2：多个独立的高可信 N2 学习资料

- 至少两个互相独立的可靠来源共同支持同一知识点。
- 优先选择专业出版社教材、日语教育机构、大学课程资料、权威教师编写资料。
- 两个互相转载的网页只算一个来源；无作者、无出处的聚合页不能作为验证来源。
- 网络资料可以用于检索线索，但必须记录页面标题、URL、访问日期和具体支持内容。

### P3：明确的日语语言规则

- 可由权威日语词典、语法辞典、语料库或语言学参考资料明确解释。
- 词汇搭配应能说明搭配限制和自然语境；文法应能说明意义、接续、语用和易混形式；阅读知识点应对应官方测量目标。
- 仅凭“听起来像日语”或 AI 判断不算验证。

### P4：AI 来源

- AI 只能生成候选题和变体，不能作为知识点的独立验证来源。
- 只有 AI 来源的知识点保持 `unverified`，不得推动正式扩题。

## 知识点验证标准

每个 knowledge point 至少满足以下一项，并在 `data/knowledge-point-sources.json` 登记证据：

1. 出现在官方公开样题或官方说明中。
2. 出现在多个独立、高可信的 JLPT N2 学习资料中。
3. 出现在用户提供且可使用的教材或真题中。
4. 能被权威日语语法、词汇或语料规则明确解释。

来源登记包含：

```json
{
  "knowledgePointId": "KP-GRA-MONONO-001",
  "validationStatus": "verified",
  "evidence": [
    {
      "evidenceType": "linguistic_reference",
      "sourceName": "资料名称",
      "sourceUrl": "https://example.com/reference",
      "citation": "版本、页码、条目或具体章节",
      "usagePermission": "reference_only",
      "verifiedBy": "审核人",
      "verifiedAt": "2026-06-30T00:00:00.000Z"
    }
  ],
  "reviewNotes": "与易混形式的区别已确认。",
  "lastReviewedAt": "2026-06-30T00:00:00.000Z"
}
```

`evidenceType` 允许：

- `official_public`
- `high_trust_material`
- `user_provided`
- `linguistic_reference`

`usagePermission` 必须区分：

- `reference_only`：只能用于验证，不复制原文。
- `user_authorized`：用户确认可录入约定范围。
- `publicly_reproducible`：来源明确允许复制或改编，并保存许可依据。

只有 `validationStatus = verified` 且至少有一条完整证据时，知识点才通过来源门禁。来源失效、出现冲突或审核人无法确认时恢复为 `unverified`。

## AI 出题标准

### AI 原创题

- 只能围绕已验证的 `knowledgePointIds` 生成。
- `sourceType` 使用 `ai_generated`，`sourceName` 记录生成系统或审核流程。
- 题干、阅读材料和例句必须原创，不拼接真题或教材原文。
- 每题必须有且只有一个最佳答案。
- 干扰项必须来自真实易错点：意义相近但语境不符、接续不符、搭配不自然或信息条件不满足。

### AI 变体题

- 必须基于已存在题目或已验证知识点。
- 不得只替换人名、数字而保留原题表达；必须重新检查语境、答案唯一性和难度。
- 仍使用 `sourceType = ai_generated`，`sourceName` 写明 `AI variant of <questionId>`，并加入 `AI变体` 标签。
- 若基础题来自受版权保护材料，只能抽取抽象考点重新原创，不能近似改写原文。

### 题目最低要求

- `grammar`、`vocabulary`、`reading`、`listening` 分开标记。
- 必须绑定至少一个已验证知识点。
- 必须有 `sourceType` 与 `sourceName`。
- `explanation` 必须说明正确答案为什么成立，并逐项或分类说明其他选项为什么不合适。
- 无知识点、无解析、无来源或存在多个合理答案的题不得进入正式题库。

## 真题与教材录入标准

1. 先登记来源和使用权限，再录入内容。
2. 保存书名/机构、版本、页码/题号、录入人、录入时间。
3. 官方公开样题仍要检查页面版权提示和第三方作品标记。
4. 只有参考权限时，仅记录知识点与题型依据，不把题目正文放入公开仓库。
5. 获准录入时，保持原文与项目解释明确分离，不把项目解释伪装成官方解析。
6. `official_sample`、`textbook` 或 `manual` 必须与实际来源一致，不能为了提高可信度而误标。

## 自动质量报告

运行：

```bash
npm run report:quality
```

报告写入 `reports/question-bank-quality.md`，输出：

- 每个知识点关联题数与来源证据。
- 每道题的 `sourceType` / `sourceName`。
- 只有 AI 题目来源的知识点。
- 缺少已验证来源的知识点。
- 解析长度不足的题目。
- 未解释干扰项的题目。
- 选项长度差异明显等“答案过于明显”候选。
- 选项重复等“可能多答案”候选。

严格门禁：

```bash
npm run validate:quality
```

当来源、解析或唯一答案存在阻断项时返回非零退出码。自动启发式只能筛查候选，无法证明日语语义唯一；正式发布仍需人工日语审核。

## 质量检查流程

1. **来源登记**：录入资料、权限与引用位置。
2. **知识点审核**：确认等级、意义、接续/搭配、易混点，状态改为 `verified`。
3. **候选题生成/录入**：只使用已验证知识点。
4. **结构校验**：运行 `npm run validate:bank`。
5. **自动质量报告**：运行 `npm run report:quality`。
6. **人工语言审核**：逐题检查自然度、唯一答案、干扰项、解释和难度。
7. **小规模试做**：记录答题时间、正确率、争议反馈；异常题回到候选区。
8. **严格门禁**：`npm run validate:quality` 与全部测试通过后才允许扩题发布。

## 扩充路线

### 80 → 300 题：覆盖优先

- 先完成现有 30 个知识点的来源验证和 80 题解析复核。
- 根据官方 N2 项目目的建立题型覆盖矩阵，补齐当前缺少的汉字读音、表记、句子组成、文章文法、中篇阅读、信息检索和听力等类型。
- 每批最多 25–50 题；每批独立审核、试做和报告，不一次生成 220 题。

### 300 → 1000 题：平衡与校准

- 每个核心知识点建立基础、标准、易混和综合四类题。
- 使用真实学习记录校准难度、作答时间、干扰项有效性和多答案投诉。
- 建立题目修订历史；有实质修改时递增 `version`，保留旧 ID 的迁移记录。

### 1000 → 3000 题：规模化治理

- 引入双人复核或专家抽检、来源失效巡检、重复/近重复检测和题目退役机制。
- 按等级、题型、知识点、难度和来源分层抽样，避免某一生成模板过度集中。
- AI 只负责候选生成、相似度检查和报告汇总；来源验证与最终发布权保持人工负责。

任何阶段只要严格质量门禁为 `HOLD`，就暂停新增正式题，先处理来源和质量债务。
